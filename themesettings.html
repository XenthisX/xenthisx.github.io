CREATING SETTINGS-BASED ANDROID THEMES

By Eli Bosley posted June 4th, 2016

When I set out to make an application I wasn’t quite sure how difficult it was going to be to get the functionality that I wanted. I knew that I would run into different issues along the way but I wasn’t sure what would cause me issues and what would not, and creating dynamic themes caused me a lot more problems than I had anticipated that it would.


The main issue in my case was one of understanding. My Java knowledge was relatively weak (at least when it comes to full-scale application development) and my Android knowledge was close to none. This, combined with the way that Android handles activities and loading of styles made creating a working theme system difficult, so I wanted to create a page that makes the process easy to follow and understand, especially for newcomers.


My main goal in creating this article is to get more developers onboard with adding themability to their apps. I appreciate unified color schemes in some situations, but a lot of apps don't support themes (even big apps like Chrome) that I wish were more customizable. This will be partially dealt with when Android N is released with Night Mode, but even beyond that having customizable action-bars and styles is very nice.


I've divided this tutorial into a few easy steps..

Step 1

Understand how Android loads themes

Android has a great function called setTheme that sets the styles for the View. This functions basically is responsible for loading the correct stylesheet when it is called. Unfortunately, it doesn't actually call any redraw functions, so any changes it makes are not applied until the app is restarted or the current view (window) is closed.

Step 2

Creating a settings activity to allow changing of settings

This part is pretty easy. Basically you just need to create a settings-fragment for every setting that you might want to change.

Step 3

Create a function that loads themes when their onCreate method is called

This is necessary so that you can redraw a window and have the theme update. The method is called on the creation of every window and checks to see what theme should currently be loaded, then applies it.

2. Create a Settiing Page

Let's create a settings page to manage the theme setting we will choose. This is easy, just right click on the app folder, mouse over to New -> Activity -> Empty Activity

Just name it SettingsActvity and leave the rest of the settings the default for now

Go ahead and open up the SettingsActivity.java file that was just created when you hit finish, and add these two things to it:

In the onCreate method, at the end:

getFragmentManager().beginTransaction().replace(android.R.id.content, new MyPreferenceFragment()).commit();

And as a separate class entirely:

    public static class MyPreferenceFragment extends PreferenceFragment {
        @Override
        public void onCreate(final Bundle savedInstanceState) {
            super.onCreate(savedInstanceState);
            addPreferencesFromResource(R.xml.preferences);
        }
    }
}

The top line is responsible for loading the preference fragment class that you created with the second snippet, and the bottom one loads a new XML file that must be created called preferences.xml. We'll create that next..

@Override

Adding the Styles and Settings 

Now that all of the setup steps are done, we can finally create a settings item and add the styles that go along with it. For the purposes of this tutorial I'm going to keep it simple and add only 1 option with a dark or a light theme, but once you get the hang of styles and Preference Fragments it takes very little time to add more.


Go to the "preferences.xml" file in your app/res/xml folder and add a list preference that looks like this:

<ListPreference
        android:defaultValue="LightTheme"
        android:entries="@array/pref_theme_titles"
        android:entryValues="@array/pref_theme_values"
        android:key="themeSelection"
        android:summary="This preference lets you choose a theme for the app"
        android:title="Select Theme" />

Again, this nests between the starting and ending PreferenceScreen tags. This file will end up looking like this:

In order to fix the errors above we must add the actual arrays that hold the items in the Preference Fragment. This is done through the auto-generated strings.xml file that is located in app/res/values/. Navigate to this file and add the following inside of the resources tags.

This top one is in charge of the displayed names for each theme

<string-array name="pref_theme_titles">
        <item>Light"</item>
        <item>Dark"</item>
    </string-array>

And the bottom string array is the actual setting item they reference, which can be grabbed from the sharedPreferenceManager.

    <string-array name="pref_theme_values">
        <item>"LightTheme"</item>
        <item>"DarkTheme"</item>
    </string-array>

Changing Colors

If you've made it this far you now have an Android app with a menu-button that allows you to change a setting, but nothing happens when the setting switches. This is dealt with through a custom java class that deals with changing theme on page loads and also with parsing the theme switches.

Make a new class called SetupTheme.java by right clicking the java -> com.yourcompany.appname folder.

Previous Next

Fill this in with the following: 

public class SetupTheme {
    public static void setupWindowTheme(Activity activity) {
        SharedPreferences pref = PreferenceManager
                .getDefaultSharedPreferences(activity);
        String themeName = pref.getString("themeSelection", "Theme");
        switch (themeName) {
            case "LightTheme":
                activity.setTheme(R.style.LightTheme);
                break;
            case "DarkTheme":
                activity.setTheme(R.style.DarkTheme);
                break;
        }
    }
}

This class can be called and given a view, and will apply the currently selected theme (read from the settings fragment you created earlier) to decide what color scheme to give the app. Right now you have errors on both of the R.style.* items because they haven't yet been created, but creating them is very easy.

Creating Styles

Now naviagate to your styles.xml file (also autogenerated) located inside app/res/values.


Inside of this is the base application theme, which is fine to leave if you want your app to boot with a different theme than you can select. For our purposes we will change it so that it is the light theme, and the dark theme is applied separately. 

Update this file so that it has these two styles inside of it, one for the light theme and one for the dark theme:

Light:

 <!-- Base application theme. -->
    <style name="LightTheme" parent="Theme.AppCompat.Light.DarkActionBar">
        <!-- Customize your theme here. -->
        <item name="colorPrimary">@color/colorPrimary</item>
        <item name="colorPrimaryDark">@color/colorPrimaryDark</item>
        <item name="colorAccent">@color/colorAccent</item>
    </style>

Dark

    <style name="DarkTheme" parent="Theme.AppCompat">
        <!-- Customize your theme here. -->
        <item name="colorPrimary">@color/colorPrimary</item>
        <item name="colorPrimaryDark">@color/colorPrimaryDark</item>
        <item name="colorAccent">@color/colorAccent</item>
    </style>

This file should now contain both of these items and your errors should be gone. Unfortunately themes don't work quite yet as we must update the app to launch with the LightTheme and add the SetupTheme function to every page we want to theme.


To fix the recently broken AppManifest issue, navigate to app/manifests then to AndroidManifest.xml. Just change the line that is red and says android:theme="@style/AppTheme" to instead say android:theme="@style/LightTheme"

Putting it All Together

Getting all of the moving parts to cooperate is easy, just add this line to the onCreate method before the view is set on any file that you want to support your new themes: 

SetupTheme.setupWindowTheme(this);

Unfortunately, we're not quite done. This is when I thought I had all of the stuff working together, but as you'll see if you try to run this the functionality is a little bit broken. You have to quit out of the app before it's even possible to see the main window's theme update, and the settings window theme doesn't update until you reopen settings. All of this is pretty easy to fix, so here goes...


In order to get the preference change to update the main window on the back button press, we need to add the override method for back button presses inside of the SettingsActivity.java file in the SettingsActivity class. This looks like this: 

    @Override
    public void onBackPressed() {
        Intent i = new Intent(getApplicationContext(), MainActivity.class);
        startActivity(i);
        finish();
    }

This will redraw the main window when back is pressed, successfully applying the theme to it.

Next, if you desperately want the theme to apply the second the theme is changed you need to add a listener that will redraw the settings window when a setting changes. This requires you to implement SharedPreferenceChangeListener in your preferenceFragment class like this:

public static class MyPreferenceFragment extends PreferenceFragment implements SharedPreferences.OnSharedPreferenceChangeListener

This requires an Override method as well: onSharedPreferenceChange, which you can add below the onCreate method in the class. That looks like this:

        @Override
        public void onSharedPreferenceChanged(SharedPreferences sharedPreferences, String key)         {
            System.out.println("key was: " + key);
            if (key != null) {
                if (getActivity() != null) {
                    getActivity().recreate();
                }
            }
        }

Finally, you need to add this line to the PreferenceFragment's class onCreate method, which registers the new listener

PreferenceManager.getDefaultSharedPreferences(this.getActivity()).registerOnSharedPreferenceChangeListener(this);

And that's it, now you have a themeable Android app that you can do whatever you would like with.
